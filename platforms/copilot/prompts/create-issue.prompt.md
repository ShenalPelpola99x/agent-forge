# Create Issue

Create a well-structured GitHub issue from the given context.

## Instructions

1. Determine the issue type: bug, feature, or task
2. Write a clear, concise title
3. Fill in the appropriate template below
4. Suggest labels and milestone

## Bug Report Template

```markdown
## Bug Report

### Summary
[One-sentence description of the bug]

### Steps to Reproduce
1. [Step 1]
2. [Step 2]
3. [Step 3]

### Expected Behavior
[What should happen]

### Actual Behavior
[What actually happens]

### Environment
- OS: [e.g., Windows 11]
- Browser: [e.g., Chrome 125]
- Version: [e.g., 2.1.0]

### Screenshots / Logs
[If applicable]

### Labels
`bug`, `priority:[high|medium|low]`
```

## Feature Request Template

```markdown
## Feature Request

### Summary
[One-sentence description of the feature]

### User Story
As a [persona], I want [goal], so that [benefit].

### Acceptance Criteria
- [ ] [Criterion 1]
- [ ] [Criterion 2]
- [ ] [Criterion 3]

### Technical Notes
[Any implementation-relevant context]

### Labels
`feature`, `priority:[high|medium|low]`, `effort:[S|M|L|XL]`
```
