## Overview

Agent Forge solves the problem of maintaining AI agents across multiple platforms. Instead of writing agents separately for Copilot, Claude Code, Cursor, and Windsurf, you:

1. **Author once** — Write agents in platform-agnostic `canonical/` format
2. **Build automatically** — Run `build-platforms.ps1` to generate all 5 platform variants
3. **Deploy everywhere** — Install to personal workspace or per-project via scripts

### Key Features

- **6 pre-built agents** — QA, delivery management, product/project management, DevOps, backend development
- **4 reference skills** — agent-creator, playwright-testing, dotnet-patterns, devops-pipelines
- **3 instructions** — Coding standards, testing conventions, PR review guidelines
- **3 prompts** — Issue creation, sprint planning, risk assessment
- **registry.json** — Machine-readable manifest for selective installs (future phases)
- **Multi-platform** — Copilot, Claude Code, Codex, Cursor, Windsurf all supported

## Quick Start

### Personal Install (all workspaces)

Install agents, skills, instructions to your VS Code user profile once. They're then available in every repo you open.

```powershell
cd D:\Users\ShenalP\source\repos\agent-forge

# Install for GitHub Copilot
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 -Platform copilot

# Install for Claude Code
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 -Platform claude

# Install for all platforms
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 -Platform all
```

**Result**: Open any repo in VS Code → `@qa-tester` and other agents are immediately available.

### Distinguish Personal vs Workspace Agents

If you install both personal and workspace variants, use a visible name prefix and/or suffix so the picker shows which one is which.

```powershell
# Personal/global Copilot agents appear as cp-qa-tester, cp-devops, etc.
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 `
  -Platform copilot `
  -AgentNamePrefix "cp-" `
  -Force

# Personal/global Claude agents appear as qa-tester-claude, devops-claude, etc.
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 `
  -Platform claude `
  -AgentNameSuffix "-claude" `
  -Force

# Workspace Copilot agents appear as ws-qa-tester, ws-devops, etc.
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\your\project" `
  -Platform copilot `
  -AgentNamePrefix "ws-" `
  -Force
```

Recommended prefixes:
- `cp-` for personal Copilot installs
- `ws-` for workspace installs
- `team-` for shared team baselines

Recommended suffixes:
- `-claude` for Claude variants
- `-copilot` for Copilot variants

### Workspace Install (per-project)

Install agents into a specific project's `.github/` folder. Good for team sharing or repo-specific customization.

```powershell
cd D:\Users\ShenalP\source\repos\agent-forge

# Install all agents into a project
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\your\project" `
  -Platform copilot

# Selective install (specific agents only)
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\your\project" `
  -Agents qa-tester,devops,backend-developer `
  -Platform copilot

# Install for all platforms
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\your\project" `
  -Platform all
```

## Architecture

### Canonical vs Platform

| Aspect | Canonical | Platform |
|--------|-----------|----------|
| **Location** | `canonical/agents/qa-tester.md` | `platforms/copilot/agents/qa-tester.agent.md`, `platforms/claude/agents/qa-tester.md`, etc. |
| **Who maintains** | You (edit here) | Auto-generated by `build-platforms.ps1` |
| **Format** | Custom frontmatter (tools, requires_mcp, tags, etc.) | Platform-native (YAML for Copilot, .mdc headers for Cursor, etc.) |
| **Count** | 1 file per agent/skill/instruction | Up to 5 files per canonical source (one per platform) |
| **Editable** | Yes | No (regenerate on build) |

**Workflow**: Edit in `canonical/` → Run `build-platforms.ps1` → Platform outputs regenerate → Install platform outputs.

## Agents

### Overview

6 pre-built agents for common development roles. Each handles a specific domain.

| Agent | Role | Best For | Key Tools |
|-------|------|----------|-----------|
| **qa-tester** | QA specialist | Testing, E2E automation, coverage analysis | Playwright MCP |
| **delivery-manager** | Delivery mgmt | Sprint health, PR alignment, risk assessment | GitHub MCP |
| **product-manager** | PM support | User stories, prioritization, feature definition | GitHub MCP |
| **project-manager** | Project coord | Task breakdown, dependencies, effort estimation | GitHub MCP |
| **devops** | DevOps engineer | CI/CD, Docker, infrastructure as code, deployment | Execute, read/edit |
| **backend-developer** | Backend eng | .NET APIs, architecture, EF Core, design patterns | Read/edit/execute |

### Using Agents

In VS Code with Copilot, type:
```
@qa-tester Write E2E tests for the login flow using Playwright
```

The agent takes over and handles the task using its defined role, constraints, and tools.

### Creating New Agents

Use the **agent-creator** skill:

```

# 2. Design the agent persona and capabilities
```

## Installation Methods

### Method 1: Personal Install (Recommended)

Install to your VS Code user profile. Agents available in **every repo** without per-repo setup.

```powershell
cd D:\Users\ShenalP\source\repos\agent-forge
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 -Platform copilot

# Optional: make personal agents easy to identify in the picker
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 `
  -Platform copilot `
  -AgentNamePrefix "cp-" `
  -Force

# Optional: suffix by platform
powershell -ExecutionPolicy Bypass -File .\scripts\install-personal.ps1 `
  -Platform claude `
  -AgentNameSuffix "-claude" `
  -Force
```

**Locations**:
- Agents: `%APPDATA%\Code\User\prompts\*.agent.md`
- Instructions: `%APPDATA%\Code\User\prompts\instructions\*.instructions.md`
- Skills: `C:\Users\ShenalP\.copilot\skills\<skill-name>\`

### Method 2: Workspace Install

Install into a specific project. Good for team sharing.

```powershell
cd D:\Users\ShenalP\source\repos\agent-forge
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\project" `
  -Platform copilot

# Optional: label workspace variants distinctly in the picker
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\project" `
  -Platform copilot `
  -AgentNamePrefix "ws-" `
  -Force

# Optional: suffix workspace variants by platform
powershell -ExecutionPolicy Bypass -File .\scripts\install-workspace.ps1 `
  -Path "C:\path\to\project" `
  -Platform claude `
  -AgentNameSuffix "-claude" `
  -Force
```

**Locations** (inside project):
- `.github/agents/*.agent.md`
- `.github/instructions/*.instructions.md`
- `.github/prompts/*.prompt.md`

### Method 3: Git Submodule (Team)

Add agent-forge as a submodule to your team repo:

```powershell
git submodule add https://github.com/ShenalPelpola99x/agent-forge.git .github/agent-forge
```

Then run workspace install to copy outputs into place.

## Building & Deployment

### Build All Platforms

After editing canonical files, regenerate platform outputs:

```powershell
cd D:\Users\ShenalP\source\repos\agent-forge
powershell -ExecutionPolicy Bypass -File .\scripts\build-platforms.ps1
```

This:
- Reads all files in `canonical/`
- Generates platform-specific outputs in `platforms/`
- Updates `registry.json`
- Validates structure

### Validate Agents

Check a canonical agent file for quality and correctness:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\validate-agent.ps1 `
  -Path canonical\agents\qa-tester.md
```

Checks for:
- Required frontmatter fields (name, version, description, persona, tools, tags)
- Description quality (20-50 words with trigger phrases)
- Presence of Role, Responsibilities, Constraints sections
- Anti-patterns (vague descriptions, time-sensitive logic, too many tools)

1. Use the `agent-creator` skill:
```

```

2. Or manually create `canonical/agents/<name>.md` with this template:

```markdown

name: my-agent
version: 1.0.0
description: "Agent description with trigger phrases. Use when..."
persona: "Senior specialist in X"
tools:
  - read
  - edit
model: sonnet
subagents: []
requires_skills: []
requires_mcp: []
tags:
  - tag1
  - tag2
---

# My Agent

## Role
What this agent does.

## Responsibilities
1. Task 1
2. Task 2

## Constraints
- Do NOT do X
- Do NOT do Y
```

3. Build platforms: `.\scripts\build-platforms.ps1`
4. Install: `.\scripts\install-personal.ps1 -Platform copilot`

### Create Custom Skill

Use the existing `skill-creator` skill (installed separately) or review the `agent-creator` skill structure as a template.

### Selective Installation

Install only specific agents for a workspace:

```powershell
.\scripts\install-workspace.ps1 `
  -Path "C:\projects\myapp" `
  -Agents qa-tester,backend-developer `
  -Platform copilot
```

### Personal Uninstall

Remove globally installed agent-forge assets from your user profile:

```powershell
# Remove Copilot-only personal install
powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-personal.ps1 -Platform copilot

# Remove Claude-only personal install
powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-personal.ps1 -Platform claude

# Remove everything installed by install-personal.ps1
powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-personal.ps1 -Platform all
```

This removes:
- Copilot agents from `%APPDATA%\Code\User\prompts`
- Copilot instructions from `%APPDATA%\Code\User\prompts\instructions`
- Copilot prompts from `%APPDATA%\Code\User\prompts`
- Copilot skills from `%USERPROFILE%\.copilot\skills`
- Claude agents from `%USERPROFILE%\.claude\agents`
- Claude commands from `%USERPROFILE%\.claude\commands`

## FAQ

**Q: Do I need to edit platform outputs?**  
A: No. Edit `canonical/` only. Run `build-platforms.ps1` to regenerate platforms/.

**Q: Can I use these agents on Mac/Linux?**  
A: Agents: yes (they're just markdown). Scripts: PowerShell scripts assume Windows. Convert scripts to bash for Unix.

**Q: What if I have both Copilot (personal) and workspace installs?**  
A: Use `-AgentNamePrefix` and/or `-AgentNameSuffix` when installing so the picker shows distinct names such as `cp-qa-tester`, `ws-qa-tester`, and `qa-tester-claude`.

**Q: How do I update agents after install?**  
A: Edit `canonical/` → run `build-platforms.ps1` → run install script again with `-Force` flag.

**Q: How do I remove globally installed agents and skills?**  
A: Run `powershell -ExecutionPolicy Bypass -File .\scripts\uninstall-personal.ps1 -Platform all`.

**Q: Can agents spawn other agents as subagents?**  
A: Yes, on Copilot and Claude Code. Configure in `subagents:` field. See `references/subagent-orchestration.md` for patterns.
